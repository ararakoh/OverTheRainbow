# 모바일 인터랙션 업데이트

## 변경 목적

사후 회상 흐름의 느린 리듬은 유지하되, 화면 전환 뒤의 공백이 로딩이나 오류로 오해되지 않도록 조정했다. 새로운 문구나 질문 순서는 추가·변경하지 않았다.

## 인터랙션 변경

- 작은 라벨은 화면 진입 직후부터 1.2초 동안 천천히 나타난다. 이전처럼 보이지 않는 상태로 대기한 뒤 등장하지 않는다.
- 제목과 보조 문장은 한 글자가 아니라 짧은 구절 단위로 이어서 나타난다. 쉼표와 마침표 뒤에만 짧은 호흡을 둔다.
- 화면을 탭하면 현재 나타나는 문장이 즉시 완성된다. 효과음, 커서, 게임식 UI는 사용하지 않는다.
- 화면 컨테이너는 미세한 opacity와 배경 변화로 먼저 살아난다. 사용자는 텍스트가 완전히 보이기 전에도 전환이 진행 중임을 인지할 수 있다.
- 진행 점은 새 화면 진입 직후 부드럽게 활성화된다. 로딩 UI가 아니라 현재 장면이 바뀌었음을 알려주는 최소 신호다.
- 헤드라인, 보조 문장, 입력 영역, CTA의 기존 순서와 느린 등장 리듬은 유지한다.
- `prefers-reduced-motion` 환경에서는 모든 요소가 즉시 표시된다.

## iPhone Safari 입력 대응

- `input`, `textarea`, `select`의 글자 크기를 16px 이상으로 맞춰 입력 시 자동 확대를 피한다.
- viewport에 `viewport-fit=cover`를 적용하고, 확대 제한 속성은 사용하지 않는다.
- 모바일에서는 `100vh`를 fallback으로 두고 `100dvh`를 우선 사용한다.
- 화면 본문은 동적 뷰포트 높이에 맞춰 스크롤되며, 키보드가 열린 상태에서는 CTA가 입력 영역 뒤에 안정적으로 이어지도록 한다.

## iPhone Safari 검증 체크리스트

- [ ] 화면 전환 직후 0.5초 안에 라벨·배경·진행 점 중 하나 이상의 변화가 보인다.
- [ ] 라벨이 갑자기 나타나지 않고 약 1.2초 동안 자연스럽게 차오른다.
- [ ] 헤드라인과 CTA가 기존의 느린 리듬을 유지한다.
- [ ] 이름, 나이, 특징 입력칸과 textarea를 탭해도 Safari 자동 확대가 발생하지 않는다.
- [ ] textarea 입력 중 키보드가 올라와도 화면이 과도하게 튀거나 CTA가 화면 밖으로 고정되지 않는다.
- [ ] 세로·가로 회전 후에도 입력 영역과 CTA가 가려지지 않는다.
- [ ] 홈 인디케이터가 있는 기기에서 하단 CTA가 안전 영역과 겹치지 않는다.
- [ ] `?test=true`에서 속도 변경, Animation Skip, 화면 강제 이동이 정상 동작한다.
- [ ] 접근성 설정의 '동작 줄이기'가 켜져 있으면 요소가 즉시 표시된다.
