# Voice QA Checklist v0.1

편지·푸시·온보딩 질문을 배포하기 전 확인한다.

## 1. 기억 정확성

- [ ] 사용자 기록에서 확인된 물건, 장소, 시간, 행동, 사건이 최소 1개 있는가?
- [ ] 그 기억의 출처를 내부적으로 연결할 수 있는가?
- [ ] 기록에 없는 성격·감정·말버릇을 추가하지 않았는가?
- [ ] 다른 반려동물의 이름으로 바꾸면 어색할 만큼 구체적인가?

## 2. 말의 태도

- [ ] 사용자를 위로하거나 감정을 해석하지 않는가?
- [ ] 반려동물을 실제로 재현한다고 주장하지 않는가?
- [ ] 문장이 일을 시키거나 기록을 독려하지 않는가?
- [ ] 한국어로 소리 내 읽었을 때 번역체나 과한 문학어가 아닌가?
- [ ] `온보딩`, `모드`, `단계`, `종료` 같은 내부 시스템 용어가 화면에 보이지 않는가?

## 3. 리듬

- [ ] 한 편지에 장면이 하나인가?
- [ ] `괜찮아요`, `볼까요`, `할게요` 같은 끝맺음이 반복되지 않는가?
- [ ] 수식어보다 물건·행동·장소가 더 많이 남는가?
- [ ] 마지막 문장이 교훈이나 위로가 아니라 장면에서 끝나는가?

## 4. 민감도

- [ ] 병원, 마지막 날, 죄책감 관련 단서는 사용자가 먼저 남긴 것인가?
- [ ] 사용자가 피하고 싶을 수 있는 기억을 기본 질문으로 밀어 넣지 않았는가?
- [ ] 사후 세계·재회·용서처럼 검증할 수 없는 말을 하지 않는가?

## 5. 최종 질문

> 이 문장을 읽고 사용자가 “감동적이네”보다 “이거, 우리 애 맞는데”라고 느낄 근거가 있는가?

근거가 없다면 감정어를 더하는 대신, 사용자 기록에서 생활 데이터 하나를 더 가져온다.
