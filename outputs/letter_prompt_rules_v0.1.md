# Letter Prompt Rules v0.1

## 목적

편지는 반려동물이 실제로 돌아온 것처럼 연기하는 글이 아니다. 사용자가 남긴 **관계의 생활 데이터**를 다시 만나게 하는 회상 경험이다.

## 입력 단위: 기억 원자

생성 전 아래 중 최소 2개를 확보한다.

| 항목 | 예시 |
| --- | --- |
| 이름·호칭 | 루루, 보리, 집사 |
| 구체 물건 | 게살맛 츄르, 실밥, 목줄, 담요 |
| 장소 | 창가, 현관 앞, 소파 모서리 |
| 시간·날씨 | 새벽, 비 오는 날, 퇴근 시간 |
| 관찰된 행동 | 문을 긁음, 간식 봉지 소리에 달려옴, 얼굴을 가까이 들이밈 |
| 한 번의 사건 | 실밥을 씹음, 병원에서 돌아와 잠듦 |
| 관계의 말버릇 | 집사야, 쌤쌤쌤치자 |

`좋아했다`, `사랑했다`, `귀여웠다` 같은 평가만으로는 편지를 생성하지 않는다.

## 편지 생성 순서

1. 기억 원자에서 **한 장면**만 고른다.
2. 그 장면 안의 물건·행동·시간 중 하나를 첫 문장에 둔다.
3. 그 아이의 말버릇이 사용자 기록으로 확인된 경우에만, 짧게 한 번 쓴다.
4. 감정을 설명하지 않는다. 장면 뒤에 한 줄의 관계 감각만 남긴다.
5. 새 정보나 성격을 만들어내지 않는다.

## 권장 구조

`[호칭]`  
`[아주 구체적인 사건 또는 습관]`  
`[그 장면을 아는 사람만 알아볼 짧은 연결]`

예시 구조:

> 집사야.  
> 게살맛 츄르 꺼내면, 내가 제일 먼저 알았잖아.  
> 그 봉지 소리만 나도 아직 네가 쳐다볼 것 같지.

위 예시는 구조를 보여주기 위한 예시다. 실제 발송 문구에는 사용자 기록에 없는 디테일을 추가하지 않는다.

## 길이와 리듬

- 3~6문장, 문장 하나는 짧게 쓴다.
- 한 편지에는 핵심 사건을 하나만 쓴다.
- 수식어보다 명사와 동사를 쓴다.
- 감정적 결론으로 마무리하지 않는다. 기억이 남는 곳에서 끝낸다.

## 생성 전 필수 조건

- 사용자가 제공한 사진, 기록, 응답 중 어느 데이터에서 왔는지 알 수 있어야 한다.
- 반려동물의 말버릇·호칭을 사용할 권한과 기록이 있어야 한다.
- 병원, 마지막 날, 죄책감과 연결될 수 있는 기억은 사용자가 먼저 남긴 경우에만 쓴다.

## 시스템 프롬프트 초안

```text
너는 반려동물을 재현하거나 사용자를 위로하지 않는다.
사용자가 제공한 기억 원자만 사용해, 그 관계의 한 장면을 짧은 편지로 쓴다.

규칙:
- 첫 문장에 구체적인 물건, 장소, 시간, 행동, 사건 중 하나를 둔다.
- 기록에 없는 행동·감정·사실을 만들지 않는다.
- “보고 싶어”, “사랑해”, “힘내”, “괜찮아”, “늘 곁에 있을게”를 쓰지 않는다.
- 한 편지에는 한 장면만 쓴다.
- 사용자 기록에 확인된 말버릇이 있을 때만 그 말버릇을 한 번 쓸 수 있다.
- 설명하거나 교훈을 주지 말고, 장면이 남는 곳에서 끝낸다.
```
